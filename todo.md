# 燕中数字母港 - V2.0 开发待办池 (To-Do List)

## 🗄️ 模块一：数据底层重构与全站联动 (Data & Sync)
> 当前痛点：城市、院校等信息混在标签字符串中以 `|` 分隔，导致地图解析极易出错，且用户无法独立维护各项属性。
- [ ] **数据库字段解耦**：在 `User` 和 `WhitelistRoster` 模型中，新增独立的标准化字段，如 `city`, `university`, `major`, `industry`，替代原有的混合 `tags`。
- [ ] **重构 CSV 导入**：更新后台导入格式严格对应新字段，加入数据清洗逻辑（如剔除“市”字以适配地图 API）。
- [ ] **全站状态自动同步**：优化地图组件直接从 `city` 字段读取数据；实现用户端修改资料后，大盘数据静默同步更新。

## ✍️ 模块二：站内投稿与审核闭环 (CMS Workflow)
> 当前痛点：重度依赖外部邮件审核，流程割裂，无法在系统内形成闭环。
- [ ] **建立站内提交表单**：前台新增投稿/活动页面，提供结构化表单（标题、正文、配图）。
- [ ] **内容状态机设计**：引入 `status` 字段（如 `DRAFT`, `PENDING`, `PUBLISHED`, `REJECTED`）。
- [ ] **后台审核工作台**：新增审核模块，集中展示 `PENDING` 投稿，提供一键通过或驳回功能。

## 👤 模块三：个人中心 UI/UX 深度优化 (Profile Optimization)
> 当前痛点：个人信息界面功能单薄，输入不规范。
- [ ] **信息分类展示**：划分为基础身份、教育轨迹、职业发展和我的投稿等独立卡片。
- [ ] **交互式资料编辑**：废弃手动输入 `|` 拼接，引入标准化的级联选择器（如省份-城市列表），确保数据规范化。
- [ ] **账号安全设置**：整合修改密码、绑定邮箱、隐私设置（名录隐藏）等功能。

## 📦 模块四：自动化数据播种与迁移管道 (Automated Seeding Pipeline)
> 当前痛点：项目部署后线上数据库为空；如果采用本地数据库文件覆盖的方式，不仅难以操作且极易抹除线上真实产生的新用户数据。需要一种自动化的初始数据注入方式。
- [ ] **统一初始数据源**：在项目中建立规范的数据存放目录（如 `prisma/data/` 或 `scripts/data/`），用于统一存放整理好的初始 `csv` 或 `json` 文件（涵盖“燕中故事”、“校友名册白名单”、“大事记”等静态资料）。
- [ ] **开发幂等性 Seed 脚本**：编写高健壮性的自动导入脚本（如更新 `prisma/seed.ts`）。引入 Upsert（更新或插入）逻辑，确保多次执行不会产生重复数据，并且在导入时自动处理数据清洗与关联。
- [ ] **部署前置钩子配置**：在 `package.json` 中配置部署命令，确保在正式上线或数据大版本更新时，可以通过简单的命令（如 `npm run db:seed`）一键完成核心展示资料的无损覆盖与同步。
